Android In-App Browser


This application was developed to introduce a useful feature in many other applications that
might require the user to go off the mobile application into a web browser, without actually
leaving the mobile application, which is useful to get users to fill in feedback forms with
ease, without needing too much effort leaving the application and searching up the form, which
most will refuse to do. The user will simply click the button, and it will open up a built-in
web browser which will allow the user to access web pages from within the mobile application.

Hospital Ward System (Java)

This hospital ward system has the following classes:
- Ward.java (To hold details of wards and beds.)
- Patient.java (To hold details of current patients.)
- PastPatients.java (To hold details of past patients.)

This hospital ward system has the following features:
- A menu, allowing the user to navigate through the system.
- See information of all the wards in the hospital.
- Display a list of all current patients.
- Display an overview of all wards, including visitors and patients.
- Admit new/Discharge patients.
- Display records of past patient visits.

Language Toggling Banking Application









This banking app was developed with a built-in feature that allows
users to toggle language settings easily, allowing it to be used by
most local Singaporeans, if the mainly spoken languages are included
in the application. The user can click the toggle button at the
top-right of the app, and select the language they want to translate
to with ease, and can select the bank that they want to get in touch
with, and it will direct them to the phone app with the bank's
pre-filled.

Dice-roll game (Java)









This is a simple java program developed for 2
players to select a fixed number of rounds to roll
the dice each game, the player that wins most
rounds out of the total round chosen will win.
However, if both players won an equal number of
rounds at the end, it will be a tie. The unique
feature of this program is that it allows users to
users to input their own username, instead of
referring to them as "Player 1" and "Player 2".
